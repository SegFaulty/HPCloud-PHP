1 Tutorial: Using Stream Wrappers {#streams-tutorial}
2 ===============================
3 
4 This is an introduction to the HPCloud-PHP library. While the library is
5 large and feature-rich, this tutorial focuses on the Stream Wrapper
6 feature. (There is also a [tutorial about the object-oriented
7 library](@ref 00-tutorial).)
8 
9 ## TL;DR
10 
11 With a few lines of setup code, you can fetch objects from HP Cloud's
12 object storage using built-in PHP functions like this:
13 
14 ~~~{.php}
15 <?php
16 // Create a new file and write it to the object store.
17 $newfile = fopen('swift://Example/my_file.txt', 'w');
18 fwrite($newfile, "Good Morning!");
19 fclose($newfile);
20 
21 // Check for an object:
22 if (file_exists('swift://Example/my_file.txt')) {
23  print "Found my_file.txt." . PHP_EOL;
24 }
25 
26 // Get an entire object at once:
27 $file = file_get_contents('swift://Example/my_file.txt');
28 print 'File: ' . $file . PHP_EOL;
29 ?>
30 ~~~
31 
32 In fact, the vast majority of file and stream functions work with
33 HPCloud's `swift://` URLs.
34 
35 The rest of this tutorial explains how they work.
36 
37 ## The Setup
38 
39 The example above does not show the code necessary for initializing the
40 HPCloud stream wrapper. In this section, we will look at the necessary
41 setup code.
42 
43 ### Loading Classes
44 
45 The HPCloud PHP library is structured following SPR-0 recommendations.
46 Practically speaking, what this means is that applications that use an
47 SPR-0 autoloader may be able to automatically load the HPCloud library.
48 
49 However, we'll assume that that is not the case. We'll assume that the
50 library needs to be initialized manually.
51 
52 What we will do is first load the HPCloud Bootstrap.php file, and then
53 use the autoloader in that file to load the rest of the library:
54 
55 ~~~{.php}
56 <?php
57 require_once '/HPCloud/Bootstrap.php';
58 
59 use \HPCloud\Bootstrap;
60 Bootstrap::useAutoloader();
61 Bootstrap::useStreamWrappers();
62 ?>
63 ~~~
64 
65 The first thing the example above does is require the Bootstrap.php
66 file, which contains code necessary for initializing the HPCloud
67 library.
68 
69 Next, we call two static methods on the HPCloud::Bootstrap object:
70 
71 - Bootstrap::useAutoLoader(): This tells HPCloud to load any other
72  classes on demand. Since we use this, we don't need any more `require`
73  or `include` statements.
74 - Bootstrap::useStreamWrappers(): This tells HPCloud to register its
75  stream wrapper classes.
76 
77 In a nutshell, PHP allows libraries to map a particular URL pattern to a
78 stream wrapper. HPCloud registers the `swift://` URL prefix. So any
79 request to a URL beginning with `swift://` will be proxied through the
80 HPCloud library.
81 
82 ## Setting Up Authentication
83 
84 When working with remote HPCloud Object Storage, you must authenticate
85 to the remote system. Authentication requires the following four pieces
86 of information:
87 
88 - account: Your account ID
89 - key: Your account's secret key
90 - tenantid: The tenant ID for the services you wish to use
91 - endpoint: The endpoint URL for HPCloud's Identity Services. It usually
92  looks something like this: `https://region-a.geo-1.identity.hpcloudsvc.com:35357`
93 
94 All four of these pieces of information can be found in the **API Keys**
95 section of your [console](https://console.hpcloud.com) account.
96 
97 (Note: You can use your username and password instead of account and
98 key, but you still must supply the tenant ID. Instead of supplying
99 `account` and `key`, use `username` and `password`.)
100 
101 We are going to look at two ways to set authentication information. The
102 first is global. That means we supply it once, and all stream and file
103 functions automatically use that information. The second is to pass
104 authentication information into the stream context.
105 
106 ### Global Configuration
107 
108 Supplying global account information has two distinct advantages:
109 
110 -# It reduces the complexity of your code
111 -# It allows context-less functions like `file_exists` and `stat` to
112  work.
113 
114 But it has a disadvantage: *Only one account can be used at a time.* Since
115 that account's information is shared across all stream wrappers, they
116 all share the same account, tenant Id, and service catalog.
117 
118 If you are working on an application that needs to connect to more than
119 one account in the same request, you may find this setup imperfect for
120 your needs.
121 
122 That said, here's how we set up a global configuration:
123 
124 ~~~{.php}
125 $settings = array(
126  'account' => YOUR_ACCOUNT_NUMBER,
127  'key' => YOUR_SECRET_KEY,
128  'tenantid' => YOUR_TENANT_ID,
129  'endpoint' => IDENTITY_SERVICES_URL,
130 
131  // Instead of account/key you can use this:
132  // 'username' => YOUR_USERNAME,
133  // 'password' => YOUR_PASSWORD,
134 );
135 Bootstrap::setConfiguration($settings);
136 ~~~
137 
138 Basically, what we do above is declare an associative array of
139 configuration parameters and then tell HPCloud::Bootstrap to set these
140 as the default configuration.
141 
142 Once the above is done, all of those PHP stream and file functions will
143 just work. All you need to do is pass them `swift://` URLs, and they
144 will do the rest.
145 
146 ## The Format of Swift URLs
147 
148 Early in the tutorial we saw some swift URLs like this:
149 `swift://Example/my_file.txt` . What is this URL referencing?
150 
151 The URL above has three important parts, in the form
152 `swift://CONTAINER/OBJECT_NAME`.
153 
154 - *swift://*: This is the schema. This part of the URL tells PHP to pass
155  the request to the HPCloud stream wrapper. (Swift, by the way, is the
156  [OpenStack name for object storage](http://openstack.org/projects/storage/).
157 - *Example*: This is the *container name*. In Object Storage parlance, a
158  container is a place to store documents. One account can have lots of
159  containers, and each container can have lots of objects.
160 - *my_file.txt*: This is the object name. An object is basically the
161  same as a file.
162 
163 Swift does not support directories, but it does allow slashes in object
164 names. So `swift://Example/this/is/my/file.png' checks the container
165 *Example* for the object named `this/is/my/file.png`.
166 
167 (For power users, there are some fancy operations you can do to treat
168 Swift filename parts as if they were directories. Check out
169 HPCloud::Storage::ObjectStorage::Container.)
170 
171 ## Using Stream Contexts for Authentication
172 
173 Sometimes it is better to pass authentication information directly to
174 the stream or file function, instead of relying upon a global
175 configuration. PHP provides for this with **stream contexts**.
176 
177 Stream contexts have one major downside: Not all PHP functions accept
178 stream contexts. Here are some notable examples:
179 
180 - file_exists()
181 - is_readable()
182 - stat()
183 
184 (Basically, anything that calls the underlying `stat(3)`.)
185 
186 The advantage, though, is that each call can have its own authentication
187 data. This is good for supporting multiple accounts, and can also be
188 used to optimize long-term performance (e.g. by saving authentication
189 tokens in a database and re-using them).
190 
191 Here's how a stream context is used:
192 
193 ~~~{.php}
194 <?php
195 require_once __DIR__ . '/../src/HPCloud/Bootstrap.php';
196 
197 use \HPCloud\Bootstrap;
198 Bootstrap::useAutoloader();
199 Bootstrap::useStreamWrappers();
200 
201 $cxt = stream_context_create(array(
202  'swift' => array(
203  'account' => YOUR_ACCOUNT_NUMBER,
204  'key' => YOUR_SECRET_KEY,
205  'tenantid' => YOUR_TENANT_ID,
206  'endpoint' => IDENTITY_SERVICES_URL,
207 
208  // Instead of account/key you can use this:
209  // 'username' => YOUR_USERNAME,
210  // 'password' => YOUR_PASSWORD,
211  ),
212 ));
213 
214 print file_get_contents('swift://Example/my_file.txt', FALSE, $cxt);
215 ?>
216 ~~~
217 
218 The main difference is the creation of `$cxt` using PHP's
219 `stream_context_create()`. To fully understand this, you may want to
220 take a look at the [PHP documentation](http://us3.php.net/manual/en/book.stream.php)
221 for streams.
222 
223 ## Stream Wrapper As A File System
224 As it was noted earlier in this tutorial, swift does not support directories.
225 Instead the names of a file can be path like with a separator. For example,
226 `swiftfs://Example/path/to/my_file.txt` has a name of `path/to/my_file.txt`.
227 
228 To enable applications to use swift in a more directory like manner there is a
229 second stream wrapper with a prefix `swiftfs://`. swiftfs stands for swift file
230 system. It works in a similar manner to to the standard stream wrappers with a
231 few key differences:
232 
233 - mkdir will return TRUE is no objects start with the directory you are trying
234  to crate. Otherwise it will return FALSE.
235 - rmdir will return FALSE if any objects start with the directory prefix you are
236  trying to remove. rmdir does not allow you to remove directories with files
237  in them.
238 - Running stat on a directory that is a prefix for some objects (e.g.,
239  `swiftfs://Example/path/to/`) will see this is a prefix for a file and treat
240  it as if it were a directory.
241 
242 To use this stream wrapper instead of the standard swift one simple replace the
243 usage of `swift://` with `swiftfs://`.
244 
245 ## Summary
246 
247 This tutorial is focused on using stream wrappers to interact with your
248 HPCloud Object Storage service. We focused on configuring the
249 environment for transparently using PHP functions like `fopen()` and
250 `file_get_contents()` to work with objects in HPCloud's object storage.
251 
252 This is just one way of interoperating with the HPCloud library. For
253 more detail-oriented work, you may find the Object Oriented facilities
254 better suited. You can read [the OO tutorial](@ref oo-tutorial) to learn
255 more about that.
256 
257 Addidtionally, you may wish to learn more about the internals of the
258 stream wrapper, the main class,
259 HPCloud::Storage::ObjectStorage::StreamWrapper, is well-documented.
