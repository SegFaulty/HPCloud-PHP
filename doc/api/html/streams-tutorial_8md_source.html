00001 Tutorial: Using Stream Wrappers   {#streams-tutorial}
00002 ===============================
00003 
00004 This is an introduction to the HPCloud-PHP library. While the library is
00005 large and feature-rich, this tutorial focuses on the Stream Wrapper
00006 feature. (There is also a [tutorial about the object-oriented
00007 library](@ref 00-tutorial).)
00008 
00009 ## TL;DR
00010 
00011 With a few lines of setup code, you can fetch objects from HP Cloud's
00012 object storage using built-in PHP functions like this:
00013 
00014 ~~~{.php}
00015 <?php
00016 // Create a new file and write it to the object store.
00017 $newfile = fopen('swift://Example/my_file.txt', 'w');
00018 fwrite($newfile, "Good Morning!");
00019 fclose($newfile);
00020 
00021 // Check for an object:
00022 if (file_exists('swift://Example/my_file.txt')) {
00023   print "Found my_file.txt." . PHP_EOL;
00024 }
00025 
00026 // Get an entire object at once:
00027 $file = file_get_contents('swift://Example/my_file.txt');
00028 print 'File: ' . $file . PHP_EOL;
00029 ?>
00030 ~~~
00031 
00032 In fact, the vast majority of file and stream functions work with
00033 HPCloud's `swift://` URLs.
00034 
00035 The rest of this tutorial explains how they work.
00036 
00037 ## The Setup
00038 
00039 The example above does not show the code necessary for initializing the
00040 HPCloud stream wrapper. In this section, we will look at the necessary
00041 setup code.
00042 
00043 ### Loading Classes
00044 
00045 The HPCloud PHP library is structured following SPR-0 recommendations.
00046 Practically speaking, what this means is that applications that use an
00047 SPR-0 autoloader may be able to automatically load the HPCloud library.
00048 
00049 However, we'll assume that that is not the case. We'll assume that the
00050 library needs to be initialized manually.
00051 
00052 What we will do is first load the HPCloud Bootstrap.php file, and then
00053 use the autoloader in that file to load the rest of the library:
00054 
00055 ~~~{.php}
00056 <?php
00057 require_once '/HPCloud/Bootstrap.php';
00058 
00059 use \HPCloud\Bootstrap;
00060 Bootstrap::useAutoloader();
00061 Bootstrap::useStreamWrappers();
00062 ?>
00063 ~~~
00064 
00065 The first thing the example above does is require the Bootstrap.php
00066 file, which contains code necessary for initializing the HPCloud
00067 library.
00068 
00069 Next, we call two static methods on the HPCloud::Bootstrap object:
00070 
00071 - Bootstrap::useAutoLoader(): This tells HPCloud to load any other
00072   classes on demand. Since we use this, we don't need any more `require`
00073   or `include` statements.
00074 - Bootstrap::useStreamWrappers(): This tells HPCloud to register its
00075   stream wrapper classes.
00076 
00077 In a nutshell, PHP allows libraries to map a particular URL pattern to a
00078 stream wrapper. HPCloud registers the `swift://` URL prefix. So any
00079 request to a URL beginning with `swift://` will be proxied through the
00080 HPCloud library.
00081 
00082 ## Setting Up Authentication
00083 
00084 When working with remote HPCloud Object Storage, you must authenticate
00085 to the remote system. Authentication requires the following four pieces
00086 of information:
00087 
00088 - account: Your account ID
00089 - key: Your account's secret key
00090 - tenantid: The tenant ID for the services you wish to use
00091 - endpoint: The endpoint URL for HPCloud's Identity Services. It usually
00092   looks something like this: `https://region-a.geo-1.identity.hpcloudsvc.com:35357`
00093 
00094 All four of these pieces of information can be found in the **API Keys**
00095 section of your [management console](https://manage.hpcloud.com)
00096 account.
00097 
00098 (Note: You can use your username and password instead of account and
00099 key, but you still must supply the tenant ID. Instead of supplying
00100 `account` and `key`, use `username` and `password`.)
00101 
00102 We are going to look at two ways to set authentication information. The
00103 first is global. That means we supply it once, and all stream and file
00104 functions automatically use that information. The second is to pass
00105 authentication information into the stream context.
00106 
00107 ### Global Configuration
00108 
00109 Supplying global account information has two distinct advantages:
00110 
00111 -# It reduces the complexity of your code
00112 -# It allows context-less functions like `file_exists` and `stat` to
00113   work.
00114 
00115 But it has a disadvantage: *Only one account can be used at a time.* Since
00116 that account's information is shared across all stream wrappers, they
00117 all share the same account, tenant Id, and service catalog.
00118 
00119 If you are working on an application that needs to connect to more than
00120 one account in the same request, you may find this setup imperfect for
00121 your needs.
00122 
00123 That said, here's how we set up a global configuration:
00124 
00125 ~~~{.php}
00126 $settings = array(
00127   'account' => YOUR_ACCOUNT_NUMBER,
00128   'key' => YOUR_SECRET_KEY,
00129   'tenantid' => YOUR_TENANT_ID,
00130   'endpoint' => IDENTITY_SERVICES_URL,
00131 
00132   // Instead of account/key you can use this:
00133   // 'username' => YOUR_USERNAME,
00134   // 'password' => YOUR_PASSWORD,
00135 );
00136 Bootstrap::setConfiguration($settings);
00137 ~~~
00138 
00139 Basically, what we do above is declare an associative array of
00140 configuration parameters and then tell HPCloud::Bootstrap to set these
00141 as the default configuration.
00142 
00143 Once the above is done, all of those PHP stream and file functions will
00144 just work. All you need to do is pass them `swift://` URLs, and they
00145 will do the rest.
00146 
00147 ## The Format of Swift URLs
00148 
00149 Early in the tutorial we saw some swift URLs like this:
00150 `swift://Example/my_file.txt` . What is this URL referencing?
00151 
00152 The URL above has three important parts, in the form
00153 `swift://CONTAINER/OBJECT_NAME`.
00154 
00155 - *swift://*: This is the schema. This part of the URL tells PHP to pass
00156   the request to the HPCloud stream wrapper. (Siwft, by the way, is the
00157   [OpenStack name for object storage](http://openstack.org/projects/storage/).
00158 - *Example*: This is the *container name*. In Object Storage parlance, a
00159   container is a place to store documents. One account can have lots of
00160   containers, and each container can have lots of objects.
00161 - *my_file.txt*: This is the object name. An object is basically the
00162   same as a file.
00163 
00164 Swift does not support directories, but it does allow slashes in object
00165 names. So `swift://Example/this/is/my/file.png' checks the container
00166 *Example* for the object named `this/is/my/file.png`.
00167 
00168 (For power users, there are some fancy operations you can do to treat
00169 Swift filename parts as if they were directories. Check out
00170 HPCloud::Storage::ObjectStorage::Container.)
00171 
00172 ## Using Stream Contexts for Authentication
00173 
00174 Sometimes it is better to pass authentication information directly to
00175 the stream or file function, instead of relying upon a global
00176 configuration. PHP provides for this with **stream contexts**.
00177 
00178 Stream contexts have one major downside: Not all PHP functions accept
00179 stream contexts. Here are some notable examples:
00180 
00181 - file_exists()
00182 - is_readable()
00183 - stat()
00184 
00185 (Basically, anything that calls the underlying `stat(3)`.)
00186 
00187 The advantage, though, is that each call can have its own authentication
00188 data. This is good for supporting multiple accounts, and can also be
00189 used to optimize long-term performance (e.g. by saving authentication
00190 tokens in a database and re-using them).
00191 
00192 Here's how a stream context is used:
00193 
00194 ~~~{.php}
00195 <?php
00196 require_once __DIR__ . '/../src/HPCloud/Bootstrap.php';
00197 
00198 use \HPCloud\Bootstrap;
00199 Bootstrap::useAutoloader();
00200 Bootstrap::useStreamWrappers();
00201 
00202 $cxt = stream_context_create(array(
00203   'swift' => array(
00204     'account' => YOUR_ACCOUNT_NUMBER,
00205     'key' => YOUR_SECRET_KEY,
00206     'tenantid' => YOUR_TENANT_ID,
00207     'endpoint' => IDENTITY_SERVICES_URL,
00208 
00209     // Instead of account/key you can use this:
00210     // 'username' => YOUR_USERNAME,
00211     // 'password' => YOUR_PASSWORD,
00212   ),
00213 ));
00214 
00215 print file_get_contents('swift://Example/my_file.txt', FALSE, $cxt);
00216 ?>
00217 ~~~
00218 
00219 The main difference is the creation of `$cxt` using PHP's
00220 `stream_context_create()`. To fully understand this, you may want to
00221 take a look at the [PHP documentation](http://us3.php.net/manual/en/book.stream.php)
00222 for streams.
00223 
00224 ## Summary
00225 
00226 This tutorial is focused on using stream wrappers to interact with your
00227 HPCloud Object Storage service. We focused on configuring the
00228 environment for transparently using PHP functions like `fopen()` and
00229 `file_get_contents()` to work with objects in HPCloud's object storage.
00230 
00231 This is just one way of interoperating with the HPCloud library. For
00232 more detail-oriented work, you may find the Object Oriented facilities
00233 better suited. You can read [the OO tutorial](@ref oo-tutorial) to learn
00234 more about that.
00235 
00236 Addidtionally, you may wish to learn more about the internals of the
00237 stream wrapper, the main class,
00238 HPCloud::Storage::ObjectStorage::StreamWrapper, is well-documented.
