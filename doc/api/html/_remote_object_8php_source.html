00030 /**
00031  * A representation of an object stored in remote Object Storage.
00032  *
00033  * A remote object is one whose canonical copy is stored in a remote
00034  * object storage. It represents a local (and possibly partial) copy of
00035  * an object. (Contrast this with HPCloud::Storage::ObjectStorage::Object)
00036  *
00037  * Depending on how the object was constructed, it may or may not have a
00038  * local copy of the entire contents of the file. It may only have the
00039  * object's "metadata" (information such as name, type, modification
00040  * date, and length of the object). Or it may have all of that in
00041  * addition to the entire content of the file.

00330   /**
00331    * Get the content of this object as a file stream.
00332    *
00333    * This is useful for large objects. Such objects should not be read
00334    * into memory all at once (as content() does), but should instead be
00335    * made available as an input stream.
00336    *
00337    * PHP offers low-level stream support in the form of PHP stream
00338    * wrappers, and this mechanism is used internally whenever available.
00339    *
00340    * If there is a local copy of the content, the stream will be read
00341    * out of the content as if it were a temp-file backed in-memory
00342    * resource. To ignore the local version, pass in TRUE for the
00343    * $refresh parameter.
00344    *
00345    * If the content is coming from a remote copy, the stream will be
00346    * read directly from the underlying IO stream.
00347    *
00348    * Each time stream() is called, a new stream is created. In most
00349    * cases, this results in a new HTTP transaction (unless $refresh is
00350    * FALSE and the content is already stored locally).
00351    *
00352    * The stream is read-only.
00353    *
00354    * @param boolean $refresh
00355    *   If this is set to TRUE, any existing local modifications will be ignored
00356    *   and the content will be refreshed from the server. Any
00357    *   local changes to the object will be discarded.
00358    * @retval resource
00359    *   A handle to the stream, which is already opened and positioned at
00360    *   the beginning of the stream.
00361    */

00363

00365     // a fake IO stream.

00470   /**
00471    * Check whether there are unsaved changes.
00472    *
00473    * An object is marked "dirty" if it has been altered
00474    * locally in such a way that it no longer matches the
00475    * remote version.
00476    *
00477    * The practical definition of dirtiness, for us, is this: An object
00478    * is dirty if and only if (a) it has locally buffered content AND (b)
00479    * the checksum of the local content does not match the checksom of
00480    * the remote content.
00481    *
00482    * Not that minor differences, such as altered character encoding, may
00483    * change the checksum value, and thus (correctly) mark the object as
00484    * dirty.
00485    *
00486    * The RemoteObject implementation does not internally check dirty
00487    * markers. It is left to implementors to ensure that dirty content is
00488    * written to the remote server when desired.

00492   public function isDirty() {
00493 
00494     // If there is no content, the object can't be dirty.
00495     if (!isset($this->content)) {
00496       return FALSE;
00497     }

00499     // Content is dirty iff content is set, and it is
00500     // different from the original content. Note that
00501     // we are using the etag from the original headers.
00502     if ($this->etag != md5($this->content)) {
00503       return TRUE;
00504     }

00506     return FALSE;
