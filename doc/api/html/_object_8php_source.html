00029 /**
00030  * An object for ObjectStorage.
00031  *
00032  * The HPCloud ObjectStorage system provides a method for storing
00033  * complete chunks of data (objects) in the cloud. This class describes
00034  * such a chunk of data.
00035  *
00036  * An object has the following basic components:
00037  *
00038  * - Name: A filename (which may be pathlike, subject to OpenStack's
00039  *   pathing rules).
00040  * - Content: The content of the object.
00041  * - Content type: The MIME type of the object. Examples:
00042  *   - text/plain; charset=UTF-8
00043  *   - image/png
00044  *   - application/x-my-custom-mime
00045  * - Metadata: File attributes that are stored along with the file on
00046  *   object store.
00047  *
00048  * Objects are stored and retrieved <i>by name</i>. So it is assumed
00049  * that, per container, no more than one file with a given name exists.
00050  *
00051  * You may create Object instance and then store them in Containers.
00052  * Likewise, a Container instance can retrieve Object instances from the
00053  * remote object store.
00054  */

00120    *
00121    * OpenStack allows you to specify metadata for a file. Metadata items
00122    * must follow these conventions:
00123    *
00124    * - names must contain only letters, numbers, and short dashes. Since
00125    *   OpenStack normalizes the name to begin with uppercase, it is
00126    *   suggested that you follow this convetion: Foo, not foo. Or you

00406   /**

00408    *
00409    * @attention EXPERT: You will need to understand OpenStack
00410    * internals to use this effectively.
00411    *
00412    * Headers set here will be added to the HTTP request during save
00413    * operations. They are not merged into existing headers until
00414    * save-time.
00415    *
00416    * This provides a mechanism for adding extension headers. CORS
00417    * headers and possibly others are stored by Swift, but have no

00419    *
00420    * There are a few things to note about this mechanism:
00421    *
00422    * - Existing headers cannot be overwritten. Only new headers can be

00428    *   header to Swift, it may simply ignore it.
00429    * - The RemoteObject::headers() method provides access to all of the
00430    *   headers returned from Swift.
00431    * - Headers are merged in as they are, with no cleaning, encoding, or
00432    *   checking. You must ensure that the headers are in the proper
00433    *   format.
00434    *
00435    * @param array $headers
00436    *   An associative array where each name is an HTTP header name, and
00437    *   each value is the HTTP header value. No encoding or escaping is
00438    *   done.
