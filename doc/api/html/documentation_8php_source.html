00006  * @section about_package Overview
00007  *
00008  * <a href="http://hpcloud.com">HPCloud</a> provides public cloud
00009  * infrastructure that is business-grade, open source-based, and developer
00010  * focused. Built on <a href="http://openstack.org">OpenStack</a>, it provides
00011  * many cloud-based services that developers can take advantage of when
00012  * building robust and reliable websites.
00013  *
00014  * The HPCloud-PHP library provides PHP developers with a fully tested,
00015  * robust, and feature-rich library for working with the HPCloud services.
00016  *
00017  * @attention
00018  * Making use of this library will require that you have several pieces of
00019  * account information for your HPCloud account (or OpenStack account, if you're
00020  * using this library with non-HP OpenStack implentations):
00021  * - account ID and secret key: For cases where you want account-wide 
00022  *   authentication/authorization.
00023  * - username/password: Typically, this is the same username/password you use
00024  *   to access the HPCloud management console.
00025  * - tenant ID: This associates an account or user with a bundle of services.
00026  *   You can find this information in your management console.
00027  * - endpoint: You will need the URL to the HPCloud endpoint responsible for
00028  *   <i>authenticating users</i>. This can be found in your management
00029  *   console.
00030  *
00031  * (If you are not sure what the "HPCloud Management Console" is, head over to
00032  * http://build.hpcloud.com. There you will find some articles and videos
00033  * explaining the HPCloud structure.)
00034  *

00063  * @section intro_example_sw Basic Example: Stream Wrappers
00064  *
00065  * The super-simple stream API:
00066  *
00067  * @code
00068  * <?php
00069  * // This is only required if you don't have a PSR-0
00070  * // autoloader to do the hard work for you.
00071  * require 'HPCloud/Bootstrap.php';
00072  *
00073  * // If you aren't using a PSR-0 autoloader,
00074  * // you might want to use this:
00075  * \HPCloud\Bootstrap::useAutoloader();
00076  *
00077  * // Turn on stream wrappers.
00078  * \HPCloud\Bootstrap::useStreamWrappers();
00079  *
00080  * // Create a stream context. You can get this
00081  * // information (including tenant ID) from your
00082  * // HPCloud management console.
00083  * $cxt = stream_context_create(array(
00084  *   'username' => 'matthew.butcher@hp.com',
00085  *   'password' => 'secret',
00086  *   'tenantid' => '123456',
00087  *   'endpoint' => 'http://url.from.hpcloud.com/',
00088  * ));
00089  *
00090  *
00091  * // Get an object from the remote object storage and read it as a string
00092  * // right into $myObject.
00093  * $myObject = file_get_contents('swift://mycontainer/foo.txt', FALSE, $cxt);
00094  *
00095  * ?>
00096  * @endcode
00097  *
00098  * With stream wrapper support, you can transparently read and write files to the
00099  * HPCloud ObjectStorage service without using any fancy API at all. Use the
00100  * normal file methods like this:
00101  *
00102  *- fopen()/fclose()
00103  *- fread()/fwrite()
00104  *- file_get_contents(), stream_get_contents()
00105  *- stat()/fstat()
00106  *- is_readable()/is_writable()
00107  *- And so on (http://us3.php.net/manual/en/ref.filesystem.php).
00108  *
00109  * Learn more about this at HPCloud::Storage::ObjectStorage::StreamWrapper.
00110  *
00111  * @section intro_example_ident Basic Example: Identity Services
00112  *
00113  * Stream wrappers are nice and all, but
00114  * some of us love fancy APIs. So here's an example using the full API
00115  * to log in and then dump a list of services that are available to you:
00116  *
00117  * @code
00118  * <?php
00119  * // This is only required if you don't have a PSR-0
00120  * // autoloader to do the hard work for you.
00121  * require 'HPCloud/Bootstrap.php';
00122  *
00123  * // If you aren't using a PSR-0 autoloader,
00124  * // you might want to use this:
00125  * \HPCloud\Bootstrap::useAutoloader();
00126  *
00127  * use \HPCloud\Services\IdentityServices;
00128  *
00129  * // Create a new identity service object, and tell it where to
00130  * // go to authenticate. This URL can be found in your HPCloud
00131  * // management console.
00132  * $identity = new IdentityServices('http://get.url.from.hpcloud.com');
00133  *
00134  * // You can authenticate either with username/password (IdentityServices::authenticateAsUser())
00135  * // or as an account/secret key (IdentityServices::authenticateAsAccount()). In either
00136  * // case you can get the info you need from the management console.
00137  * $account = '123456789098765';
00138  * $secret = 'dgasgasd';
00139  * $tenantId = '56545654';
00140  *
00141  * // $token will be your authorization key when you connect to other
00142  * // services. You can also get it from $identity->token().
00143  * $token = $identity->authenticateAsAccount($account, $secret, $tenantId);
00144  *
00145  * // Get a listing of all of the services you currently have configured on
00146  * // HPCloud.
00147  * $catalog = $itentity->serviceCatalog();
00148  *
00149  * var_dump($catalog);
00150  *
00151  * ?>
00152  * @endcode
00153  *
00154  *-# Our classes use PHP namespaces to organize components. If you've never used
00155  *   them before, don't worry. They're easy to get the hang of.
00156  *-# The Bootstrap class handles setting up HPCloud services. Read about it at HPCloud::Bootstrap.
00157  *-# The IdentityServices class handles authenticating to HP, discovering services, and providing
00158  *   access to your account. HPCloud::Services::IdentityServices explains the details, but here are
00159  *   a few functions you'll want to know:
00160  *   - HPCloud::Services::IdentityServices::__construct() tells the object where to connect.
00161  *   - HPCloud::Services::IdentityServices::authenticateAsUser() lets you log
00162  *     in with username and password.
00163  *   - HPCloud::Services::IdentityServices::authenticateAsUser() lets you log
00164  *     in with account number and secret key.
00165  *   - HPCloud::Services::IdentityServices::serviceCatalog() tells you about
00166  *     the services you have activated on this account.

00168  * @section intro_example_swift Basic Example: Object Storage
00169  *
00170  * Assuming you have an object storage instance available in your service
00171  * catalog, we could continue on with something like this:
00172  *
00173  * @code
00174  * <?php
00175  * // Find out where our ObjectStorage instance lives:
00176  * $storageList = $identity->serviceCatalog('object-storage');
00177  * $objectStorageUrl = storageList[0]['endpoints'][0]['publicURL'];
00178  *
00179  * // Create a new ObjectStorage instance:
00180  * $objectStore = new \HPCloud\Storage\ObjectStorage($token, $objectStorageUrl);
00181  *
00182  * // List containers:
00183  * print_r($objectStore->containers());
00184  *
00185  * // Get a container named 'stuff':
00186  * $container = $objectStore->container('stuff');
00187  *
00188  * // List all of the objects in that container:
00189  * print_r($container->objects());
00190  *
00191  * // Get an object named 'example.txt'
00192  * $obj = $container->object('example.txt');
00193  *
00194  * // Print that object's contents:
00195  * print $obj->content();
00196  *
00197  * // Actually, since it implements __tostring, we could do this:
00198  * print $obj;
00199  * ?>
00200  * @endcode
00201  *
00202  * This shows you a few methods for accessing objects and containers on your
00203  * HPCloud::Storage::ObjectStorage account. There are many functions for
00204  * creating and modifying containers and objects, too.
00205  *
00206  *- HPCloud::Storage::ObjectStorage is where you will start.
00207  *- Container services are in HPCloud::Storage::ObjectStorage::Container
00208  *- There are two classes for objects:
00209  *     - HPCloud::Storage::ObjectStorage::Object is for creating new objects.
00210  *     - HPCloud::Storage::ObjectStorage::RemoteObject provides better network
00211  *     performance when reading objects.
