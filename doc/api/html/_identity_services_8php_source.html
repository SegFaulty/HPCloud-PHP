00030 /**
00031  * IdentityServices provides authentication and authorization.
00032  *
00033  * IdentityServices (a.k.a. Keystone) provides a central service for managing
00034  * other services. Through it, you can do the following:
00035  *
00036  * - Authenticate
00037  * - Obtain tokens valid accross services
00038  * - Obtain a list of the services currently available with a token
00039  * - Associate with tenants using tenant IDs.
00040  *
00041  * @b AUTHENTICATION
00042  *
00043  * The authentication process consists of a single transaction during which the
00044  * client (us) submits credentials and the server verifies those credentials,
00045  * returning a token (for subsequent requests), account information, and the
00046  * service catalog.
00047  *
00048  * Authentication credentials:
00049  *
00050  * - Username and password
00051  * - Account ID and Secret Key
00052  *
00053  * Other mechanisms may be supported in the future.
00054  *
00055  * @b TENANTS
00056  *
00057  * Services are associated with tenants. A token is returned when
00058  * authentication succeeds. It *may* be associated with a tenant. If it is not,
00059  * it is called "unscoped", and it will not have access to any services.
00060  *
00061  * A token that is associated with a tenant is considered "scoped". This token
00062  * can be used to access any of the services attached to that tenant.
00063  *
00064  * There are two different ways to attach a tenant to a token:
00065  *
00066  * - During authentication, provide a tenant ID. This will attach a tenant at
00067  *   the outset.
00068  * - After authentication, "rescope" the token to attach it to a tenant. This
00069  *   is done with the rescope() method.
00070  *
00071  * <b>Where do I get a tenant ID?</b>
00072  *
00073  * There are two notable places to get this information:
00074  *
00075  * A list of tenants associated with this account can be obtain programatically
00076  * using the tenants() method on this object.
00077  *
00078  * HPCloud customers can find their tenant ID in the console along with their
00079  * account ID and secret key.
00080  *
00081  * @b EXAMPLE
00082  *
00083  * The following example illustrates typical use of this class.
00084  *
00085  * @code
00086  * <?php
00087  * // You may need to use \HPCloud\Bootstrap to set things up first.
00088  *
00089  * use \HPCloud\Services\IdentityServices;
00090  *
00091  * // Create a new object with the endpoint URL (no version number)
00092  * $ident = new IdentityServices('https://example.com:35357');
00093  *
00094  * // Authenticate and set the tenant ID simultaneously.
00095  * $ident->authenticateAsUser('butcher@hp.com', 'password', '1234567');
00096  *
00097  * // The token to use when connecting to other services:
00098  * $token = $ident->token();
00099  *
00100  * // The tenant ID.
00101  * $tenant = $ident->tenantId();
00102  *
00103  * // Details about what services this token can access.
00104  * $services = $ident->serviceCatalog();
00105  *
00106  * // List all available tenants.
00107  * $tenants = $ident->tenants();
00108  *
00109  * // Switch to a different tenant.
00110  * $ident->rescope($tenants[0]['id']);
00111  *
00112  * ?>
00113  * @endcode
00114  *
00115  * <b>PERFORMANCE CONSIDERATIONS</b>
00116  *
00117  * The following methods require network requests:
00118  *
00119  * - authenticate()
00120  * - authenticateAsUser()
00121  * - authenticateAsAccount()
00122  * - tenants()
00123  * - rescope()
00124  *
00125  *
00126  */
