00032 /**
00033  * Provides stream wrapping for Swift.
00034  *
00035  * This provides a full stream wrapper to expose `swift://` URLs to the
00036  * PHP stream system.
00037  *
00038  * Swift streams provide authenticated and priviledged access to the
00039  * swift data store. These URLs are not generally used for granting
00040  * unauthenticated access to files (which can be done using the HTTP
00041  * stream wrapper -- no need for swift-specific logic).
00042  *
00043  * <b>URL Structure</b>
00044  *
00045  * This takes URLs of the following form:
00046  *
00047  * <tt>swift://CONTAINER/FILE</tt>
00048  *
00049  * Example:
00050  *
00051  * <tt>swift://public/example.txt</tt>
00052  *
00053  * The example above would access the `public` container and attempt to
00054  * retrieve the file named `example.txt`.
00055  *
00056  * Slashes are legal in Swift filenames, so a pathlike URL can be constructed
00057  * like this:
00058  *
00059  * <tt>swift://public/path/like/file/name.txt</tt>
00060  *
00061  * The above would attempt to find a file in object storage named
00062  * `path/like/file/name.txt`.
00063  *
00064  * A note on UTF-8 and URLs: PHP does not yet natively support many UTF-8
00065  * characters in URLs. Thus, you ought to rawurlencode() your container name
00066  * and object name (path) if there is any possibility that it will contain
00067  * UTF-8 characters.
00068  *
00069  * <b>Locking</b>
00070  *
00071  * This library does not support locking (e.g. flock()). This is because the
00072  * OpenStack Object Storage implementation does not support locking. But there
00073  * are a few related things you should keep in mind:
00074  *
00075  * - Working with a stream is essentially working with a COPY OF a remote file.
00076  *   Local locking is not an issue.
00077  * - If you open two streams for the same object, you will be working with
00078  *   TWO COPIES of the object. This can, of course, lead to nasty race
00079  *   conditions if each copy is modified.
00080  *
00081  * <b>Usage</b>
00082  *
00083  * The principle purpose of this wrapper is to make it easy to access and
00084  * manipulate objects on a remote object storage instance. Managing
00085  * containers is a secondary concern (and can often better be managed using
00086  * the HPCloud API). Consequently, almost all actions done through the
00087  * stream wrapper are focused on objects, not containers, servers, etc.
00088  *
00089  * <b>Retrieving an Existing Object</b>
00090  *
00091  * Retrieving an object is done by opening a file handle to that object.
00092  *
00093  * <b>Writing an Object</b>
00094  *
00095  * Nothing is written to the remote storage until the file is closed. This
00096  * keeps network traffic at a minimum, and respects the more-or-less stateless
00097  * nature of ObjectStorage.
00098  *
00099  * <b>USING FILE/STREAM RESOURCES</b>
00100  *
00101  * In general, you should access files like this:
00102  *
00103  * @code
00104  * <?php
00105  * \HPCloud\Bootstrap::useStreamWrappers();
00106  * // Set up the context.
00107  * $context = stream_context_create(
00108  *   array('swift' => array(
00109  *     'account' => ACCOUNT_NUMBER,
00110  *     'key' => SECRET_KEY,
00111  *     'tenantId' => TENANT_ID
00112  *     'endpoint' => AUTH_ENDPOINT_URL,
00113  *   )
00114  *  )
00115  * );
00116  * // Open the file.
00117  * $handle = fopen('swift://mycontainer/myobject.txt', 'r+', FALSE, $context);
00118  *
00119  * // You can get the entire file, or use fread() to loop through the file.
00120  * $contents = stream_get_contents($handle);
00121  *
00122  * fclose($handle);
00123  * ?>
00124  * @endcode
00125  *
00126  * @remarks
00127  *
00128  * - file_get_contents() works fine.
00129  * - You can write to a stream, too. Nothing is pushed to the server until
00130  *   fflush() or fclose() is called.
00131  * - Mode strings (w, r, w+, r+, c, c+, a, a+, x, x+) all work, though certain
00132  *   constraints are slightly relaxed to accomodate efficient local buffering.
00133  * - Files are buffered locally.
00134  *
00135  * <b>USING FILE-LEVEL FUNCTIONS</b>
00136  *
00137  * PHP provides a number of file-level functions that stream wrappers can
00138  * optionally support. Here are a few such functions:
00139  *
00140  * - file_exists()
00141  * - is_readable()
00142  * - stat()
00143  * - filesize()
00144  * - fileperms()
00145  *
00146  * The HPCloud stream wrapper provides support for these file-level functions.
00147  * But there are a few things you should know:
00148  *
00149  * - Each call to one of these functions generates at least one request. It may
00150  *   be as many as three:
00151  *   * An auth request
00152  *   * A request for the container (to get container permissions)
00153  *   * A request for the object
00154  * - <em>IMPORTANT:</em> Unlike the fopen()/fclose()... functions NONE of these functions
00155  *   retrieves the body of the file. If you are working with large files, using
00156  *   these functions may be orders of magnitude faster than using fopen(), etc.
00157  *   (The crucial detail: These kick off a HEAD request, will fopen() does a
00158  *   GET request).
00159  * - You must use Bootstrap::setConfiguration() to pass in all of the values you
00160  *   would normally pass into a stream context:
00161  *   * endpoint
00162  *   * account
00163  *   * key
00164  * - Most of the information from this family of calls can also be obtained using
00165  *   fstat(). If you were going to open a stream anyway, you might as well use
00166  *   fopen()/fstat().
00167  * - stat() and fstat() fake the permissions and ownership as follows:
00168  *   * uid/gid are always sset to the current user. This basically assumes that if
00169  *     the current user can access the object, the current user has ownership over
00170  *     the file. As the OpenStack ACL system developers, this may change.
00171  *   * Mode is faked. Swift uses ACLs, not UNIX mode strings. So we fake the string:
00172  *     - 770: The ACL has the object marked as private.
00173  *     - 775: The ACL has the object marked as public.
00174  *     - ACLs are actually set on the container, so every file in a public container
00175  *       will return 775.
00176  * - stat/fstat provide only one timestamp. Swift only tracks mtime, so mtime, atime,
00177  *   and ctime are all set to the last modified time.
00178  *
00179  * <b>DIRECTORIES</b>
00180  *
00181  * OpenStack Swift does not really have directories. Rather, it allows
00182  * characters such as '/' to be used to designate namespaces on object
00183  * names. (For simplicity, this library uses only '/' as a separator).
00184  *
00185  * This allows for simulated directory listings. Requesting 
00186  * `scandir('swift://foo/bar/')` is really a request to "find all of the items
00187  * in the 'foo' container whose names start with 'bar/'".
00188  *
00189  * Because of this...
00190  *
00191  * - Directory reading functions like scandir(), opendir(), readdir()
00192  *   and so forth are supported.
00193  * - Functions to create or remove directories (mkdir() and rmdir()) are
00194  *   meaningless, and thus not supported.
00195  *
00196  * Swift still has support for "directory markers" (special zero-byte files
00197  * that act like directories). However, since there are no standards for how
00198  * said markers ought to be created, they are not supported by the stream
00199  * wrapper.
00200  *
00201  * As usual, the underlying HPCloud::Storage::ObjectStorage::Container class
00202  * supports the full range of Swift features.
00203  *
00204  * <b>SUPPORTED CONTEXT PARAMETERS</b>
00205  *
00206  * This section details paramters that can be passed <i>either</i>
00207  * through a stream context <i>or</i> through
00208  * HPCloud::Bootstrap::setConfiguration().
00209  *
00210  * @attention
00211  * PHP functions that do not allow you to pass a context may still be supported
00212  * here <em>IF</em> you have set options using Bootstrap::setConfiguration().
00213  *
00214  * You are <i>required</i> to pass in authentication information. This
00215  * comes in one of three forms:
00216  *
00217  * -# API keys: acccount, key, tenantid, endpoint
00218  * -# User login: username, password, tenantid, endpoint
00219  * -# Existing (valid) token: token, swift_endpoint
00220  *
00221  * The third method (token) can be used when the application has already
00222  * authenticated. In this case, a token has been generated and assigneet 
00223  * to an account and tenant ID.
00224  *
00225  * The following parameters may be set either in the stream context
00226  * or through HPCloud::Bootstrap::setConfiguration():
00227  *
00228  * - token: An auth token. If this is supplied, authentication is skipped and
00229  *     this token is used. NOTE: You MUST set swift_endpoint if using this
00230  *     option.
00231  * - swift_endpoint: The URL to the swift instance. This is only necessary if
00232  *     'token' is set. Otherwise it is ignored.
00233  * - username: A username. MUST be accompanied by 'password' and 'tenantid'.
00234  * - password: A password. MUST be accompanied by 'username' and 'tenantid'.
00235  * - account: An account ID. MUST be accompanied by a 'key' and 'tenantid'.
00236  * - key: A secret key. MUST be accompanied by an 'account' and 'tenantid'.
00237  * - endpoint: The URL to the authentication endpoint. Necessary if you are not
00238  *     using a 'token' and 'swift_endpoint'.
00239  * - use_swift_auth: If this is set to TRUE, it will force the app to use
00240  *     the deprecated swiftAuth instead of IdentityServices authentication.
00241  *     In general, you should avoid using this.
00242  * - content_type: This is effective only when writing files. It will
00243  *     set the Content-Type of the file during upload.
00244  * - use_cdn: If this is set to TRUE, then whenever possible assets will be
00245  *     loaded from CDN instead of from the object store. The following
00246  *     conditions must obtain for this to work:
00247  *     - The container must allow public reading (ACL)
00248  *     - The container must have CDN enabled
00249  *     - The CDN container must be active ("cdn-enabled")
00250  *     - Authentication info must be accessible to the stream wrapper.
00251  * - cdn_require_ssl: If this is set to FALSE, then CDN-based requests
00252  *     may use plain HTTP instead of HTTPS. This will spead up CDN
00253  *     fetches at the cost of security.
00254  *
00255  * @attention
00256  *  ADVANCED: You can also pass an HPCloud::Storage::CDN object in use_cdn instead of
00257  *  a boolean.
00258  *
00259  * @see http://us3.php.net/manual/en/class.streamwrapper.php
00260  */

01232   /**
01233    * Generate a reasonably accurate STAT array.
01234    *
01235    * Notes on mode:
01236    * - All modes are of the (octal) form 100XXX, where
01237    *   XXX is replaced by the permission string. Thus,
01238    *   this always reports that the type is "file" (100).
01239    * - Currently, only two permission sets are generated:
01240    *   - 770: Represents the ACL::makePrivate() perm.
01241    *   - 775: Represents the ACL::makePublic() perm.
01242    *
01243    * Notes on mtime/atime/ctime:
01244    * - For whatever reason, Swift only stores one timestamp.
01245    *   We use that for mtime, atime, and ctime.
01246    *
01247    * Notes on size:
01248    * - Size must be calculated externally, as it will sometimes
01249    *   be the remote's Content-Length, and it will sometimes be
01250    *   the cached stat['size'] for the underlying buffer.
01251    */
01252   protected function generateStat($object, $container, $size) {
01253     // This is not entirely accurate. Basically, if the 
01254     // file is marked public, it gets 100775, and if
01255     // it is private, it gets 100770.
01256     //
01257     // Mode is always set to file (100XXX) because there
01258     // is no alternative that is more sensible. PHP docs
01259     // do not recommend an alternative.
01260     //
01261     // octdec(100770) == 33272
01262     // octdec(100775) == 33277
01263     $mode = $container->acl()->isPublic() ? 33277 : 33272;
01264 
01265     // We have to fake the UID value in order for is_readible()/is_writable()
01266     // to work. Note that on Windows systems, stat does not look for a UID.
01267     if (function_exists('posix_geteuid')) {
01268       $uid = posix_geteuid();
01269       $gid = posix_getegid();
01270     }
01271     else {
01272       $uid = 0;
01273       $gid = 0;
01274     }
01275 
01276     if ($object instanceof \HPCloud\Storage\ObjectStorage\RemoteObject) {
01277       $modTime = $object->lastModified();
01278     }
01279     else {
01280       $modTime = 0;
01281     }
01282     $values = array(
01283       'dev' => 0,
01284       'ino' => 0,
01285       'mode' => $mode,
01286       'nlink' => 0,
01287       'uid' => $uid,
01288       'gid' => $gid,
01289       'rdev' => 0,
01290       'size' => $size,
01291       'atime' => $modTime,
01292       'mtime' => $modTime,
01293       'ctime' => $modTime,
01294       'blksize' => -1,
01295       'blocks' => -1,
01296     );
01297 
01298     $final = array_values($values) + $values;
01299 
01300     return $final;
01301 
01302   }
