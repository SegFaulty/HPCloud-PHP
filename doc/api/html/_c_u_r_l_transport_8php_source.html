00031 /**
00032  * Provide HTTP transport with CURL.
00033  *
00034  * You should choose the Curl backend if...
00035  *
00036  * - You KNOW Curl support is compiled into your PHP version
00037  * - You do not like the built-in PHP HTTP handler
00038  * - Performance is a big deal to you
00039  * - You will be sending large objects (>2M)
00040  * - Or PHP stream wrappers for URLs are not supported on your system.
00041  *
00042  * CURL is demonstrably faster than the built-in PHP HTTP handling, so
00043  * ths library gives a performance boost. Error reporting is slightly
00044  * better too.
00045  *
00046  * But the real strong point to Curl is that it can take file objects
00047  * and send them over HTTP without having to buffer them into strings
00048  * first. This saves memory and processing.
00049  *
00050  * The only downside to Curl is that it is not available on all hosts.
00051  * Some installations of PHP do not compile support.

00236     $buffer = array();
