00031  * A container in an ObjectStorage.
00032  *
00033  * An Object Storage instance is divided into containers, where each
00034  * container can hold an arbitrary number of objects. This class
00035  * describes a container, providing access to its properties and to the
00036  * objects stored inside of it.

00275   /**

00277    *
00278    * @attention
00279    * Typically a container should be created by ObjectStorage::createContainer().
00280    * Get existing containers with ObjectStorage::container() or
00281    * ObjectStorage::containers(). Using the constructor directly has some
00282    * side effects of which you should be aware.
00283    *
00284    * Simply creating a container does not save the container remotely.
00285    *
00286    * Also, this does no checking of the underlying container. That is, simply
00287    * constructing a Container in no way guarantees that such a container exists
00288    * on the origin object store.
00289    *

00291    * and one of its accessors is called before the accessed values are initialized, then

00293    *
00294    * Containers loaded from ObjectStorage::container() or Container::newFromRemote()

00296    * transaction to fetch properties.

00298    * The practical result of this:
00299    *
00300    * - If you are creating a new container, it is best to do so with
00301    *   ObjectStorage::createContainer().

00307    *   manipulating the container directly will likely involve an extra HTTP

00310    *   option.

00834   /**
00835    * Retrieve a list of Objects with the given prefix.
00836    *
00837    * Object Storage containers support directory-like organization. To
00838    * get a list of items inside of a particular "subdirectory", provide
00839    * the directory name as a "prefix". This will return only objects
00840    * that begin with that prefix.
00841    *
00842    * (Directory-like behavior is also supported by using "directory
00843    * markers". See objectsByPath().)
00844    *
00845    * Prefixes
00846    *
00847    * Prefixes are basically substring patterns that are matched against
00848    * files on the remote object storage.
00849    *
00850    * When a prefix is used, object storage will begin to return not just
00851    * Object instsances, but also Subdir instances. A Subdir is simply a
00852    * container for a "path name".
00853    *
00854    * Delimiters
00855    *
00856    * Object Storage (OpenStack Swift) does not have a native concept of
00857    * files and directories when it comes to paths. Instead, it merely
00858    * represents them and simulates their behavior under specific
00859    * circumstances.
00860    *
00861    * The default behavior (when prefixes are used) is to treat the '/'
00862    * character as a delimiter. Thus, when it encounters a name like
00863    * this: `foo/bar/baz.txt` and the prefix is `foo/`, it will
00864    * parse return a Subdir called `foo/bar`.
00865    *
00866    * Similarly, if you store a file called `foo:bar:baz.txt` and then
00867    * set the delimiter to `:` and the prefix to `foo:`, it will return
00868    * the Subdir `foo:bar`. However, merely setting the delimiter back to
00869    * `/` will not allow you to query `foo/bar` and get the contents of
00870    * `foo:bar`.
00871    *
00872    * Setting $delimiter will tell the Object Storage server which
00873    * character to parse the filenames on. This means that if you use
00874    * delimiters other than '/', you need to be very consistent with your
00875    * usage or else you may get surprising results.
00876    *
00877    * @param string $prefix
00878    *   The leading prefix.
00879    * @param string $delimiter
00880    *   The character used to delimit names. By default, this is '/'.
00881    * @param int $limit
00882    *   An integer indicating the maximum number of items to return. This
00883    *   cannot be greater than the Swift maximum (10k).
00884    * @param string $marker
00885    *   The name of the object to start with. The query will begin with
00886    *   the next object AFTER this one.
00887    * @retval array
00888    *   List of RemoteObject or Subdir instances.
00889    */

01023    * Perform the HTTP query for a list of objects and de-serialize the
01024    * results.

01027     if (isset($limit)) {
01028       $params['limit'] = (int) $limit;
01029       if (!empty($marker)) {
01030         $params['marker'] = (string) $marker;
01031       }
01032     }
01033 
01034     // We always want JSON.
01035     $params['format'] = 'json';
01036 
01037     $query = http_build_query($params);
01038     $query = str_replace('%2F', '/', $query);
01039     $url = $this->url . '?' . $query;
01040 
01041     $client = \HPCloud\Transport::instance();
01042     $headers = array(
01043       'X-Auth-Token' => $this->token,
01044     );
01045 
01046     $response = $client->doRequest($url, 'GET', $headers);
01047 
01048     // The only codes that should be returned are 200 and the ones
01049     // already thrown by doRequest.
01050     if ($response->status() != 200) {
01051       throw new \HPCloud\Exception('An unknown exception occurred while processing the request.');
01052     }
01053 
01054     $responseContent = $response->content();
01055     $json = json_decode($responseContent, TRUE);
01056 
01057     // Turn the array into a list of RemoteObject instances.
01058     $list = array();

01060       if (!empty($item['subdir'])) {
01061         $list[] = new Subdir($item['subdir'], $params['delimiter']);
01062       }
01063       elseif (empty($item['name'])) {
01064         throw new \HPCloud\Exception('Unexpected entity returned.');
01065       }
