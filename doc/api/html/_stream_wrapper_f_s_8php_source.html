00022 /**
00023  * @file
00024  * Contains the stream wrapper for `swiftfs://` URLs.
00025  * 
00026  * <b>Note, this stream wrapper is in early testing.</b>
00027  * 
00028  * The stream wrapper implemented in HPCloud\Storage\ObjectStorage\StreamWrapper
00029  * only supports the elements of a stream that are implemented by object
00030  * storage. This is how the PHP documentation states a stream wrapper should be
00031  * created. Because some features do not exist, attempting to treat a stream
00032  * wrapper as if it were a file system will not entirely work. For example,
00033  * while there are not directories objects have pathy names (with / separators).
00034  * Directory calls to object storage with the default stream wrappers will not
00035  * operate how they would for a file system.
00036  * 
00037  * StreamWrapperFS is an attempt to make a filesystem like stream wrapper.
00038  * Hence the protocol is swiftfs standing for swift file system.
00039  * 
00040  * To understand how this stream wrapper works start by first reading the
00041  * documentation on the HPCloud::Storage::ObjectStorage::StreamWrapper.
00042  * 
00043  * <b>DIRECTORIES</b>
00044  * 
00045  * Because OpenStack Swift does not support directories the swift:// stream
00046  * wrapper does not support them. This stream wrapper attempts to fake them by
00047  * faking directory stats, mkdir, and rmdir. By default (see the options below
00048  * for how to change these) directories have permissions of 777, timestamps
00049  * close to that of the request, and the user and group called by php. We mock
00050  * these on the fly though information is stored in the PHP stat cache.
00051  * 
00052  * In addition to the parameters supported by StreamWrapper, the following
00053  * parameters may be set either in the stream context or through 
00054  * HPCloud::Bootstrap::setConfiguration():
00055  * - swiftfs_fake_stat_mode: Directories don't exist in swift. When stat() is
00056  *     is called on a directory we mock the stat information so functions like
00057  *     is_dir will work. The default file permissions is 0777. Though this
00058  *     parameter you can pass is a different set of file permissions to use
00059  *     for these mock stats.
00060  * - swiftfs_fake_isdir_true: Directory functions like mkdir and is_dir (stat)
00061  *     check to see if there are objects with the the passed in directory as a
00062  *     prefix to see if it already exists. If you want is_dir to always return
00063  *     true even if it is not an existing prefix set this to TRUE. Defaults to
00064  *     FALSE.
00065  */
